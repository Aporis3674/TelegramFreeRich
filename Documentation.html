TelegramFreeRich

Design Specification — v2.0 (Corrected)
Last updated: July 2026

A desktop WYSIWYG rich text editor for Telegram Bot API 10.1.

Send structured, formatted messages through any Telegram bot. No Premium required.

1. What

A desktop WYSIWYG editor for Telegram rich messages. User types visually, sees a live preview, and sends via sendRichMessage (Bot API 10.1, June 2026). No code visible. No Premium subscription needed.

2. Audience

Anyone who wants to send professionally formatted Telegram messages without paying for Premium. Telegram locked rich text editing to Premium users, but Bot API 10.1 gives bots the same capability for free. This app bridges that gap.

3. Architecture (CRITICAL CHANGE)

OLD (Wrong)

contenteditable DOM → Markdown string → sendRichMessage(markdown)

The real API does not accept a Markdown string for complex blocks. It expects a structured JSON array of InputRichBlock* objects.

NEW (Correct)

contenteditable DOM → Block State (JSON[]) → sendRichMessage(blocks)

Each message section is an independent block object in an array. The editor must maintain this structured state from the start -- not try to extract blocks from a flat string later.

Block Structure Example

[
  { "type": "heading", "level": 2, "text": "Title" },
  { "type": "paragraph", "text": "Text with **bold** and _italic_" },
  { "type": "code_block", "language": "js", "text": "console.log('hi')" },
  { "type": "table", "cells": [["Col A", "Col B"], ["1", "2"]] },
  { "type": "list", "style": "bullet", "items": ["Item 1", "Item 2"] },
  { "type": "details", "summary": "Expand", "content": "Hidden content" },
  { "type": "media", "media_type": "photo", "url": "..." },
  { "type": "divider" },
  { "type": "footer", "text": "Footnote 1" }
]

Each block maps to its equivalent InputRichBlock* class at send time.

4. Critical: Checklist is Separate from Rich Message

FeatureAPISend Method
Text, lists, tables, media, math, footnotes, etc.Rich MessagesendRichMessage with InputRichMessage
Checklist (done/todo)ChecklistsendChecklist with InputChecklist

UI implication: When the user clicks "Checklist", the app must call a separate API (sendChecklist), not include it in the Rich Message block array. If the user wants both rich text and a checklist in one "message", two separate API calls must be made. This must be visually clear in the UI.

Toolbar - Three Visual Groups

  1. Inline: Bold, Italic, Underline, Strikethrough, Code, Spoiler, Highlight, Subscript, Superscript
  2. Block: Bullet List, Numbered List, Checklist (visually distinct - separate API), Link, Code Block, Table, Collapsible, Divider, Footnote
  3. Media & Special: Image, Map, Slideshow, Collage, Math

Block-Based Editor

Instead of raw contenteditable, each block is an independent card with a drag handle -- similar to Notion / EditorJS pattern. This matches the real API structure (block array) and enables drag-to-reorder for tables, slideshows, and lists.

Live Preview

Right panel, Telegram-style message bubble:

  • Rounded corners, light blue background
  • Hover on any section shows a subtle border for that block
  • All formatting rendered inline

Settings Panel (Gear Icon)

  • Bot Token (from @BotFather)
  • Chat ID (channel or group)
  • "Test Connection" button -- calls getMe before any real send
  • Send method: sendRichMessage (recommended), sendRichMessageDraft, editMessageText
