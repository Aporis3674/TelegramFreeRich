# TelegramFreeRich — سند طراحی (نسخه‌ی تصحیح‌شده)

## ۱. چیه؟

یه ویرایشگر متن پیشرفته (WYSIWYG) برای تلگرام. کاربر می‌نویسه، می‌بینه، ارسال می‌کنه — بدون دیدن هیچ کدی. از طریق Bot API تلگرام (متد واقعی `sendRichMessage`، اضافه‌شده در Bot API 10.1) پیام غنی (Rich Message) می‌فرسته.

## ۲. مخاطب

هر کسی که می‌خواد پیام حرفه‌ای و قالب‌بندی‌شده توی تلگرام بفرسته — بدون نیاز به اشتراک Premium. این قابلیت برای کاربران عادی فقط از طریق بات در دسترسه، برای همین این اپ ارزش داره.

---

## ۳. معماری (مهم‌ترین تغییر نسخه‌ی قبل)

### قبل (اشتباه)
```
contenteditable DOM → Markdown string → sendRichMessage(markdown)
```
مشکل: API واقعی تلگرام متن مارک‌داون نمی‌خواد؛ یه آرایه از بلوک‌های JSON ساختاریافته می‌خواد.

### بعد (درست)
```
contenteditable DOM → Block State (JSON[]) → sendRichMessage(blocks)
```

هر بخش از پیام یه **بلوک مستقل** توی یه آرایه‌ست. ویرایشگر باید از همون لحظه‌ی اول این state رو نگه داره، نه اینکه بعداً سعی کنه از یه رشته‌ی متن، بلوک استخراج کنه.

### نمونه‌ی ساختار بلوک‌ها

```json
[
  { "type": "heading", "level": 2, "text": "عنوان" },
  { "type": "paragraph", "text": "متن پاراگراف با **بولد** و _ایتالیک_" },
  { "type": "code_block", "language": "js", "text": "console.log('hi')" },
  { "type": "table", "cells": [["ستون A", "ستون B"], ["۱", "۲"]] },
  { "type": "list", "style": "bullet", "items": ["آیتم ۱", "آیتم ۲"] },
  { "type": "details", "summary": "باز کن", "content": "محتوای پنهان" },
  { "type": "media", "media_type": "photo", "url": "..." },
  { "type": "divider" },
  { "type": "footer", "text": "پانویس شماره ۱" }
]
```

هر بلوک هنگام ارسال به شکل کلاس معادل خودش در API تبدیل می‌شه (مثلاً `InputRichBlockTable`, `InputRichBlockList`, `InputRichBlockDetails` و ...).

---

## ۴. نکته‌ی حیاتی: چک‌لیست جدا از Rich Message است

| ویژگی | API | متد ارسال |
|---|---|---|
| متن، لیست، جدول، رسانه، ریاضی، پانویس و... | Rich Message | `sendRichMessage` با `InputRichMessage` |
| چک‌لیست (انجام‌شده / انجام‌نشده) | Checklist | `sendChecklist` با `InputChecklist` |

نتیجه برای UI: وقتی کاربر دکمه‌ی «چک‌لیست» رو می‌زنه، اپ باید در پس‌زمینه بفهمه این یه بلوک جدا از بقیه‌ی پیام‌هاست و موقع ارسال، یه فراخوانی API متفاوت (`sendChecklist`) صدا بزنه — نه اینکه سعی کنه چک‌لیست رو داخل آرایه‌ی بلوک‌های `sendRichMessage` جا بده. اگه کاربر هم متن غنی و هم چک‌لیست توی یه «پیام» بخواد، عملاً باید دو پیام جدا برای تلگرام ارسال بشه (یکی Rich Message، یکی Checklist) و این باید به‌وضوح توی UI مشخص باشه، نه یه ارسال واحد.

---

## ۵. اصلاح نام‌های تکنیکال

| نام قبلی در طرح | نام واقعی در API |
|---|---|
| هایلایت | `Marked` (`RichTextMarked`) |
| اسپویلر | `Spoiler` (`RichTextSpoiler`) |
| کدبلاک | `pre` (`InputRichBlockPreformatted`) |
| کد درون‌خطی | `Code` (`RichTextCode`) |

این‌ها فقط برای هماهنگی بین اسم دکمه‌ی UI و نام واقعی توی کد مهمن؛ کاربر همون برچسب فارسیِ آشنا («هایلایت»، «اسپویلر») رو می‌بینه.

---

## ۶. قابلیت‌های پشتیبانی‌شده (لیست نهایی)

**متن درون‌خطی (Inline):** بولد، ایتالیک، زیرخط، خط‌خورده، اسپویلر، هایلایت (Marked)، زیرنویس، بالانویس، کد

**لیست:** گلوله‌ای، شماره‌دار
*(چک‌لیست جدا — بخش ۴ را ببین)*

**بلوک:** سرتیتر H1–H6، نقل‌قول، نقل‌قول برجسته (Pull Quote)، بلوک کد با انتخاب زبان، خط جداکننده، تاشو (Details)

**جدول:** با سرستون

**رسانه:** عکس، ویدیو، صدا، اسلایدشو، کلاژ، نقشه

**ریاضیات:** فرمول درون‌خطی و بلوکی

**پانویس:** با شماره‌گذاری خودکار و لینک ارجاع (Anchor / Reference Link)

## ۷. رابط کاربری

پنجره اصلی دو قسمتی:

```
┌─────────────────────────────────────────────┐
│  [T] TelegramFreeRich          [_] [O] [X]  │
├──────────────────┬──────────────────────────┤
│                  │                          │
│   نوار ابزار    │     پیش‌نمایش زنده       │
│ ┌──────────────┐ │   ┌──────────────────┐   │
│ │ B I U S C ...│ │   │  حباب پیام       │   │
│ └──────────────┘ │   │  تلگرام (شبیه چت)│   │
│                  │   └──────────────────┘   │
│   ویرایشگر      │                          │
│   بلوک‌محور     │   [ارسال]  [ویرایش]      │
│  (کارت‌های قابل  │                          │
│   جابه‌جایی)     │                          │
├──────────────────┴──────────────────────────┤
│  شمارشگر کاراکتر: ۱۲۰۰/۳۲۷۶۸              │
│  [تنظیمات]                                  │
└─────────────────────────────────────────────┘
```

### نوار ابزار — سه گروه بصری با جداکننده

1. **درون‌خطی:** بولد، ایتالیک، زیرخط، خط‌خورده، کد، اسپویلر، هایلایت، زیرنویس، بالانویس
2. **بلوک:** لیست گلوله‌ای، لیست شماره‌دار، چک‌لیست *(با نشان بصری متفاوت چون API مجزاست)*، لینک، کد بلند، جدول، تاشو، خط جداکننده، پانویس
3. **رسانه و ویژه:** تصویر، نقشه، اسلایدشو، کلاژ، ریاضیات

### ویرایشگر بلوک‌محور

به‌جای `contenteditable` خام، هر بلوک یه کارت مستقل با دستگیره‌ی جابه‌جایی (drag handle) است — شبیه الگوی Notion / EditorJS. این هم با معماری واقعی API هم‌خون‌تره و هم برای جابه‌جا کردن جدول، اسلایدشو یا لیست خیلی کاربردی‌تره.

### پیش‌نمایش زنده

سمت راست، دقیقاً شبیه حباب پیام تلگرام:
- گوشه‌های گرد، پس‌زمینه‌ی آبی کم‌رنگ
- هاور روی هر بخش، مرز همون بلوک رو با یه حاشیه‌ی کم‌رنگ نشون می‌ده تا کاربر بفهمه هر تیکه از کجا تا کجاست

### پنل تنظیمات (دکمه‌ی چرخ‌دنده)

- توکن بات (از @BotFather)
- شناسه‌ی چت (کانال یا گروه)
- دکمه‌ی **«تست اتصال»** — یه فراخوانی `getMe` قبل از ارسال واقعی، تا کاربر مطمئن بشه توکن و چت آیدی درسته

## ۸. ویژگی‌های فنی

- **ابزار:** Electron (Node.js + Chromium)
- **فرانت:** HTML + CSS + JavaScript
- **مدل داده:** Block State (آرایه‌ای از آبجکت‌های JSON) — نه رشته‌ی مارک‌داون
- **خروجی:** Block State → تبدیل به کلاس‌های `InputRichBlock*` → `sendRichMessage`
  - در صورت وجود چک‌لیست: فراخوانی جداگانه‌ی `sendChecklist`
- **تم:** تاریک (پیش‌فرض) + روشن
- **میانبرها:** Ctrl+B، Ctrl+I، Ctrl+U، Ctrl+K، Ctrl+E
- **قابل نصب:** Windows، macOS، Linux

---

## ۹. سوالات باز برای مرحله‌ی بعد

- آیا کاربر باید بتونه پیام رو قبل از ارسال به‌صورت پیش‌نویس ذخیره کنه (با استفاده از `sendRichMessageDraft`)؟
- برای اسلایدشو و کلاژ، محدودیت تعداد و حجم رسانه چقدره؟
- آیا نیاز به پشتیبانی از چند زبان (فارسی/عربی/انگلیسی) در همون رابط کاربری هست، یا فقط فارسی؟
