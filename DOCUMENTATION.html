1. Project Concept

Telegram introduced a rich text editor in June 2026, but restricted it to Premium users only. However, Bot API 10.1 gives bots the exact same capabilities for free. This project creates a visual WYSIWYG editor that lets any user — not just Premium subscribers — create and send professionally formatted messages.

The best things in life are free. The second-best things are also free, if you use a bot.

What Telegram does with Premium

  • Rich text editor with headings, tables, code blocks, collages, etc.
  • Only available in the official Telegram app
  • Requires Telegram Premium subscription

What this project does

  • Same rich text features — available to everyone for free
  • WYSIWYG editor in a browser (no Telegram required)
  • Sends via Bot API 10.1 — sendRichMessage with markdown field
  • Also supports editMessageText with rich_message

2. Architecture

Core Pipeline

contenteditable DOM
    ↓  (DOM Parser: convertDOMToMarkdown)
  Markdown string
    ↓  (sendRichMessage)
Telegram API 10.1
    ↓
  Rendered rich message in chat

Key insight: The markdown field of sendRichMessage supports the most features with the least complexity. No need for complex blocks parsing — Markdown handles everything: inline formatting, headings, lists, tables, code blocks, math, details, footnotes, media, slideshows, and maps.

3. Bot API 10.1 Methods

3.1 sendRichMessage

Sends a rich message with advanced formatting.

POST https://api.telegram.org/bot{TOKEN}/sendRichMessage

{
  "chat_id": "@channel or user_id",
  "rich_message": {
    "markdown": "...",   // ← USE THIS
    "html": "...",       // or this (fewer features)
    "blocks": [...]      // or this (complex)
  }
}

TESTED All three fields work. markdown recommended for maximum features.

3.2 sendRichMessageDraft

Sends a temporary 30-second preview (for AI streaming). Must be finalized with sendRichMessage.

{
  "chat_id": "user_id",
  "rich_message": { "markdown": "..." },
  "draft_id": 12345
}

PRIVATE CHAT ONLY

3.3 editMessageText + rich_message

Edits a previously sent message using rich formatting.

POST https://api.telegram.org/bot{TOKEN}/editMessageText

{
  "chat_id": "@channel",
  "message_id": 123,
  "rich_message": {
    "markdown": "*Edited* text!"
  }
}

TESTED Works perfectly.

3.4 sendMessage (legacy fallback)

{
  "chat_id": "@channel",
  "text": "<b>bold</b>",
  "parse_mode": "HTML"
}

Simple formatting only. No footnotes, no details, no media. Use only as fallback.

9. Recommended Strategy

The Plan: DOM → Markdown → sendRichMessage

  1. User writes in WYSIWYG editor (contenteditable div)
  2. JS converts DOM nodes to Markdown string
  3. Send via sendRichMessage with markdown field
  4. Fallback: editMessageText + rich_message for edits
  5. Fallback: sendMessage + parse_mode=HTML for simple cases

11. UI/UX Design

Layout

  • Desktop: Split pane — Editor (60%) + Live Preview (40%)
  • Tablet: 50/50 split
  • Mobile: Stacked — Editor top, Preview bottom (toggle)

Theme

  • Default: Dark theme (Telegram-inspired colors)
  • Optional: Light theme toggle
  • CSS variables for all colors

Typography

  • System fonts: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif
  • Code: 'Fira Code', 'JetBrains Mono', monospace
  • Base size: 16px desktop, 14px mobile

Icons

  • All SVG — no emoji, no icon fonts
  • Minimal, clean design

Features

  • Floating toolbar with formatting buttons
  • Plus menu for block insertion
  • Settings panel (Bot Token, Chat ID, Send Method)
  • Live Telegram-style preview
  • Drag-and-drop media
  • Keyboard shortcuts (Ctrl+B, I, U, K, E)
  • Focus mode (hide all UI)
  • Clear All button
  • Character counter (32,768 max)
